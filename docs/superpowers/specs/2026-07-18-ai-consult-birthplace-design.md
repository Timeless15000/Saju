# AI 상담 연동 + 출생지 입력 설계 (2026-07-18)

## 목표
1. 사주 상담소·종합운세를 규칙 기반 → Claude AI 전문가 분석으로 업그레이드
2. 출생지 입력(검색 자동완성)으로 진태양시 보정을 도시별 경도 기준으로 정확하게

## 결정 사항 (사용자 확인 완료)
- AI 연결: 사용자가 본인 Anthropic API 키를 앱 설정에 입력 → 브라우저에서 Claude API 직접 호출 (서버 없음)
- 출생지: 입력하면 자동완성되는 검색형 (내장 도시 DB: 한국 25곳 + 해외 40곳)
- AI 적용 범위: 상담소 채팅 + 종합운세 심층 분석 (둘 다)

## 구현
### 출생지 / 진태양시
- 도시 DB: `{이름, 국가, 경도, IANA 시간대, 남반구 여부}` 내장. `<datalist>` 자동완성.
- 시간대 오프셋: `Intl.DateTimeFormat(timeZone)` 으로 출생 시점의 UTC 오프셋 계산 (오프라인, 1954~61 한국 UTC+8:30·서머타임 자동 처리)
- 진태양시 보정분 = round(경도×4 − 오프셋분). 서울 −32, 부산 −24, 시드니 +5 등.
- 엔진: `computeSaju`에 `tzOffsetMin`(기본 540), `lmtCorrMin`(기본 −32) 파라미터 추가 — 기존 테스트 26건과 하위 호환.
- 안내: 사주는 태어난 곳 기준, 현재 거주지는 원국에 영향 없음. 남반구 계절 반전 논쟁은 남반구 출생자에게만 해당.

### AI 연동
- 설정 카드(푸터 링크로 열림): API 키(localStorage) + 모델 선택(Opus 4.8 기본 / Sonnet 5 / Haiku 4.5)
- 호출: `fetch https://api.anthropic.com/v1/messages` + `anthropic-dangerous-direct-browser-access: true`, SSE 스트리밍, adaptive thinking(Haiku 제외)
- 상담소: 사주 전체 데이터(원국·지장간·오행·신강약·용신·십신·대운·향후 10년 세운·나이·성별·출생지)를 system 프롬프트에 담아 대화 이력과 함께 전송. 스트리밍 표시. 키 없거나 오류 시 규칙 기반 폴백.
- 종합운세: "AI 전문가 심층 분석" 섹션 — 같은 사주는 localStorage 캐시, 재생성 버튼.
